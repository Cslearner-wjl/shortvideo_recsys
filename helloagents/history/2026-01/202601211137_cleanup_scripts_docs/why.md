# 变更提案: 脚本注释与文档迁移整理

## 需求背景
现有脚本分散且用途不清晰，README 中包含大量启动/运行说明不易检索。需要为脚本补充用途与启动方式说明，将“生成视频”目录合并至 scripts，并将启动类文档集中到 docs 以提升可维护性。

## 变更内容
1. 为所有 `.sh`/`.ps1` 脚本补充开头注释（用途+启动命令）。
2. 将 `生成视频/` 目录迁移到 `scripts/` 并统一整理。
3. 将 README 中启动/运行/依赖说明迁移至 `docs/` 并增加清晰标识；README 保留入口与链接。
4. 在 docs 中补充 `scripts/crawl_and_import.sh` 的用途说明与使用方式。

## 影响范围
- **模块:** scripts, docs, README
- **文件:** 根目录与 `scripts/`、`bigdata/`、`backend/` 下的脚本；`docs/*`；`README.md`
- **API:** 无
- **数据:** 无

## 核心场景

### 需求: 脚本说明补全
**模块:** scripts
为 `.sh`/`.ps1` 脚本增加统一的用途与启动命令注释，便于快速识别与执行。

#### 场景: 脚本用途快速识别
执行打开脚本文件时可直接看到用途描述与启动命令。
- 预期结果: 每个脚本文件开头包含用途与启动命令说明

### 需求: 目录整理
**模块:** scripts
将 `生成视频/` 目录迁移至 `scripts/` 并保持功能可用。

#### 场景: 生成视频目录合并
所有生成视频相关内容位于 `scripts/生成视频/` 下。
- 预期结果: 目录合并完成且路径清晰

### 需求: 文档迁移与标识
**模块:** docs
将启动后端、前端、依赖等命令迁移到 docs，并提供可检索入口。

#### 场景: 启动指南集中
打开 docs 即可定位后端/前端/依赖启动步骤与脚本说明。
- 预期结果: docs 有明确入口与小节指引，README 保留跳转

## 风险评估
- **风险:** 脚本注释位置不一致或误改执行命令导致执行失败；目录迁移导致路径引用失效。
- **缓解:** 仅在文件头部插入注释且不修改原有命令；迁移后更新文档与引用路径。
