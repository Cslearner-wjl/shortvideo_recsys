# 技术设计: 脚本注释与文档迁移整理

## 技术方案
### 核心技术
- Bash / PowerShell 脚本注释规范
- Markdown 文档重组
- 文件移动与路径更新

### 实现要点
- 盘点仓库内所有 `.sh`/`.ps1` 脚本文件，在文件开头补充统一注释：
  - `用途:` 说明脚本目的
  - `用法:` 给出从仓库根目录执行的命令
- 迁移 `生成视频/` 到 `scripts/`，保持原文件不改名；如有引用路径，统一更新到新位置。
- 将 README 中“依赖启动/后端启动/前端启动/常见排查/脚本说明”等内容迁移到 docs，新增清晰目录与索引。
- README 保留简短入口与链接，指向 docs 中的启动指南与脚本说明。
