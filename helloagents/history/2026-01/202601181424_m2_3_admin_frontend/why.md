# 变更提案: M2.3 管理端前端（Vue3 + ECharts）

## 需求背景
当前管理端仅有空页面骨架，缺少管理员登录、用户与视频管理、数据看板等核心能力，无法支撑内容审核与运营分析。

## 产品分析

### 目标用户与场景
- **用户群体:** 管理员/运营
- **使用场景:** 登录后台后进行用户管理、视频审核与运营数据查看
- **核心痛点:** 无统一入口、操作分散、缺少可视化数据

### 价值主张与成功指标
- **价值主张:** 提供统一、可落地的管理端工作台
- **成功指标:** 登录与核心页面可用；列表支持分页/搜索/操作；图表稳定渲染；无新增后端能力

### 人文关怀
避免展示非必要敏感信息，清晰提示操作后果（冻结/删除/审核），降低误操作风险。

## 变更内容
1. 管理员登录页与鉴权流程
2. 用户管理页（分页/搜索/冻结解封）
3. 视频管理页（列表/审核/设热门/删除/上传）
4. 数据看板页（播放趋势、用户增长、TopN 热门、活跃度）
5. 路由、表格组件、ECharts 图表组件与 API client

## 影响范围
- **模块:** frontend-admin
- **文件:** frontend/admin/src/**
- **API:** /api/auth/login, /api/admin/users, /api/admin/videos, /api/admin/analytics/*, /api/videos/page
- **数据:** 不新增后端数据模型，仅消费现有接口

## 核心场景

### 需求: 管理员登录
**模块:** frontend-admin
支持管理员输入账号密码登录，登录成功后进入后台主页。

#### 场景: 基础登录
输入账号密码后完成登录并跳转到数据看板。
- 登录失败时提示原因

### 需求: 用户管理
**模块:** frontend-admin
支持分页列表、关键词搜索以及冻结/解封操作。

#### 场景: 用户搜索与状态更新
输入关键词查询用户列表，点击冻结或解封并得到明确提示。
- 操作后列表状态刷新

### 需求: 视频管理
**模块:** frontend-admin
支持视频列表、审核、设热门、删除与上传。

#### 场景: 视频审核与运营
审核视频、设热门或删除后列表状态更新，上传视频支持填写标题与描述。
- 操作失败时提示原因

### 需求: 数据看板
**模块:** frontend-admin
展示播放趋势、用户增长、TopN 热门、活跃度。

#### 场景: 数据总览
图表在进入页面后加载渲染，支持空数据展示提示。

## 风险评估
- **风险:** 管理端视频列表无专用接口，可能需复用 `/api/videos/page`
- **缓解:** 前端兼容基础字段，必要时在联调说明中标注数据来源与限制
