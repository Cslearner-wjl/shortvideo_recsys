# 变更提案: 管理端黑灰风格与用户端推荐/榜单增强

## 需求背景
当前管理端与用户端界面风格分散、视觉层级与操作反馈不足，用户端推荐/榜单功能缺少筛选排序、空态和连续浏览体验，评论列表 API 也无法满足两端分页与字段诉求。需要统一视觉与交互体验，并补齐关键数据能力。

## 产品分析

### 目标用户与场景
- **用户群体:** 平台运营人员（管理端），内容消费者（用户端）
- **使用场景:** 管理端日常运营与审核；用户端内容浏览、互动与探索
- **核心痛点:** 管理端风格不一致、信息密度与可读性不足；用户端筛选和榜单信息不足、失败反馈不清晰

### 价值主张与成功指标
- **价值主张:** 统一黑灰极简风格提升专业感与可用性，增强推荐/榜单与评论能力提升用户可读性与互动效率
- **成功指标:** 管理端四页风格一致；推荐/榜单支持时间范围与热度/播放/点赞排序；评论列表分页可用且字段满足双端需求；失败操作可弹窗重试

### 人文关怀
确保交互反馈明确，避免误触；评论列表仅展示必要信息，管理端字段遵循最小可见原则。

## 变更内容
1. 管理端登录/用户管理/视频管理/看板统一黑灰极简风与更清晰信息层级
2. 用户端推荐/榜单页新增筛选/排序/封面/作者/统计指标/无限滚动/空态
3. 视频卡片内字段栅格对齐（标题/作者/播放量/点赞/评论/收藏）
4. 播放/互动失败提示统一为弹窗并支持手动重试
5. 新增评论列表 API：支持分页、时间排序、双端字段差异与评论互动字段

## 影响范围
- **模块:** 管理端前端、用户端前端、后端视频/评论模块、知识库
- **文件:** 前端页面与样式、后端 controller/service/mapper/dto、wiki/api.md、wiki/data.md、wiki/modules
- **API:** 新增评论列表接口；扩展评论点赞/互动数据
- **数据:** comments 表关联字段、评论互动统计字段

## 核心场景

### 需求: 管理端黑灰极简风
**模块:** frontend/admin
管理端登录/用户管理/视频管理/看板统一视觉与交互规范

#### 场景: 管理端页面访问
运营人员进入任一页面时
- 页面保持统一黑灰主题、层级清晰
- 表格与卡片视觉一致，操作按钮易识别

### 需求: 用户端推荐/榜单能力增强
**模块:** frontend/user
推荐与榜单页支持筛选排序、统计信息、连续滚动与空态

#### 场景: 用户筛选与浏览
用户筛选时间范围、排序后
- 列表可刷新并无限滚动
- 空态有明确信息与引导

### 需求: 视频卡片字段对齐
**模块:** frontend/user
卡片内标题/作者/播放量/点赞/评论/收藏按固定栅格对齐

#### 场景: 用户浏览卡片
- 信息列对齐，易于对比
- 不影响移动端布局

### 需求: 播放/互动失败弹窗重试
**模块:** frontend/user
播放/点赞/收藏/评论失败弹窗提示

#### 场景: 操作失败
- 弹窗展示失败原因
- 可手动重试触发操作

### 需求: 评论列表 API
**模块:** backend/video
新增评论列表接口，分页按时间排序，双端字段差异

#### 场景: 用户端/管理端获取评论
- 用户端字段精简、含互动信息
- 管理端字段含审核与用户标识
