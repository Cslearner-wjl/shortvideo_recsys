短视频分析推荐系统需求规格说明书（SRS）
1 引言
1.1 目的
本需求规格说明书用于描述“短视频分析推荐系统”的业务需求与功能边界，为后续的系统设计、开发实现、测试验收提供依据，主要作用包括：
    1.	明确系统需求与功能范围，统一产品、研发、测试对项目目标的理解；
    2.	为开发人员进行概要设计、详细设计、编码提供依据；
    3.	为测试人员编写测试用例及验收标准提供依据；
    4.	明确数据采集需求（用户行为、视频数据、推荐数据），为推荐算法与分析报表提供支撑。

1.2 背景
随着短视频内容爆发式增长，用户在平台上的浏览行为更加碎片化，对内容的匹配效率要求更高。
本系统旨在通过采集用户行为（播放、点赞、评论、收藏）与视频内容特征，实现：
    1. 	短视频内容管理
    2.	热门榜单统计
    3.	用户个性化推荐
    4.	行为数据采集与分析看板
从而提高内容分发效率与用户留存。

2 项目概述
2.1 系统目标
系统面向两类使用者：
    •	普通用户（用户前端）：登录注册、浏览短视频、点赞评论收藏、热门榜单、个性化推荐；
    •	管理员（管理后台）：用户管理、视频管理（上传/审核/热门设置/删除）、数据分析看板。
系统主要目标：
    1.	支持短视频播放与互动行为记录；
    2.	提供热门视频排行；
    3.	提供基于用户行为的个性化推荐；
    4.	提供管理端数据统计分析能力。

2.2 系统架构（逻辑结构）
系统整体采用 前后端分离架构：
    •	用户前端：Vue3 + Echarts
    •	管理后台：Vue3+ Echarts
    •	后端应用服务
1.	Java + SpringBoot（SpringMVC + Spring + MyBatisPlus）
2.	负责提供 REST API，处理用户、视频、行为数据、推荐结果等业务逻辑
    •	对象存储：MinIO：存储视频文件与封面
    •	数据库：MySQL：业务数据存储（用户、视频、评论、点赞等）
    •	推荐与统计：可基于行为表实现协同过滤 / 热度计算（初期可用规则、热门推荐）

2.3 用户角色定义

普通用户:
    描述:注册并使用短视频浏览推荐功能	
    权限范围:浏览视频、互动行为、推荐、榜单
管理员:
	描述：后台管理视频与用户，查看统计数据	
    权限范围：审核视频、设置热门、用户管理、数据看板

3 系统功能需求
3.1 用户前端需求（web页面、参考web端抖音）

3.1.1 登录页面
功能描述： 用户使用账号密码登录系统进入首页。
•	输入字段：用户名/邮箱/手机号 + 密码
•	校验规则：
    1.	必填校验
    2.	登录失败提示：账号（先）或密码（后）错误
•	成功行为：跳转至首页
3.1.2 注册页面
功能描述： 新用户注册账号。
•	输入字段：用户名、手机号、邮箱、密码、邮箱验证码
•	校验规则：
    o	手机号必须为 11 位数字
    o	邮箱格式正确且唯一
    o	验证码有效期 5 分钟
    o	注册名唯一
•	行为：
    o	注册成功跳转登录页面
    o	注册失败弹出提示信息
3.1.3 浏览短视频
功能描述： 用户可刷短视频，系统记录播放行为。
    •	支持上下滑/列表播放
    •	视频播放时触发播放行为采集
    •	视频信息展示：
        o	标题、作者、播放量、点赞数、评论数、收藏数
3.1.4 用户互动功能
用户可对短视频执行互动行为：
    1.	点赞
    2.	评论
    3.	收藏
    4.	播放
系统必须记录每一次互动行为，用于推荐与统计。
3.1.5 热门排行榜
功能描述： 按热度指标展示热门短视频榜单。
    •	默认按热度分数降序
    •	热度指标可由：播放量、点赞数、评论数、收藏数加权计算
    •	支持分页加载或滚动加载
3.1.6 个性化推荐
功能描述： 根据用户历史行为推荐短视频。
•	输入数据来源：
    o	用户行为记录（播放、点赞、评论、收藏）
    o	视频热度/标签特征
•	输出：
    o	推荐视频列表
•	推荐逻辑（可分阶段实现）：
    o	阶段1：热门+随机+行为偏好（规则）
    o	阶段2：协同过滤（UserCF / ItemCF）

3.2 管理后台需求

3.2.1 普通用户管理
后台管理员可对用户进行管理：
    •	查看用户列表（分页）
    •	冻结账号 / 解封账号
    •	用户信息查询（按账号/用户名/手机号/邮箱精确查找）
3.2.2 视频管理
后台管理员可对视频执行生命周期管理：
1.	上传短视频
    o	上传后存储到 MinIO
    o	保存视频信息到数据库
2.	审核短视频
    o	审核状态：待审核 / 通过 / 驳回
3.	设为热门
    o	热门标记字段
4.	删除
    o	删除数据库记录 + MinIO 对象
3.2.3 数据分析看板
后台提供数据统计与展示能力（Echarts）：
•	离线数据分析统计图
    o	每日播放量趋势
    o	用户增长趋势
    o	热门视频 TopN
•	数据分析看板（管理员可视化）
    o	用户活跃度
    o	视频发布量
    o	推荐点击率（可选，如果推荐点击率到达一定程度，可申请“优秀创作者”）
3.2.4 后台管理员账号管理
支持管理员账号信息维护：
    •	管理员登录
    •	管理员信息管理（新增/修改/删除）
    •	管理员密码修改

3.3 应用服务（后端接口需求）
后端主要提供三类服务接口：
3.3.1 用户管理服务
    •	用户注册
    •	用户登录
    •	修改信息
    •	冻结/解封
3.3.2 短视频服务
    •	视频上传（MinIO）
    •	视频审核
    •	视频删除
    •	设置热门
    •	视频分页查询（按热度/时间）
3.3.3 用户行为采集服务
采集行为类型包含：
    •	播放
    •	点赞
    •	评论
    •	收藏
行为采集要求：
    •	必须记录：用户ID、视频ID、行为类型、行为时间
    •	播放行为可扩展：播放时长、是否完播
3.3.4 推荐短视频服务
推荐服务提供两类接口：
1.	推荐列表（个性化）
2.	热门视频列表

4 约束
1.	前端技术栈：Vue3 + Echarts
2.	后端技术栈：SpringBoot + SpringMVC + MyBatisPlus
3.	视频存储：MinIO
4.	数据库：MySQL
5.	推荐算法：初期可规则推荐，后期扩展协同过滤/深度学习模型
